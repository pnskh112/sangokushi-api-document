DB設計について

  • 情報はデータと文脈を合成して生まれる
    システムとは、このデータと情報の区別という観点から見た場合、
    ユーザーがシステムにデータを登録し、さらにそれを引き出して情報を作り出す
    という一連のサイクルの中に位置付けることもできる
@startuml
title 負傷部位・原因登録画面表示のシーケンス図

skinparam ParticipantPadding 20
skinparam BoxPadding 10

actor "タブレットユーザ" as user

box "Tablet" #LightBlue
    participant "タブレットアプリ" as pwa
end box

box "AWS" #LightBlue
    participant "EC2" as ec2
    participant "RDS" as db
    database "S3" as s3
end box

activate user
    user -> pwa: 負傷部位・原因登録ボタンを押下
    activate pwa
        pwa -> ec2: 治療計画情報一覧を取得 <b>*1</b>

        activate ec2
            ec2 -> db:　対象テーブルから情報を取得
            activate db
                db --> ec2: データ返却
            deactivate
            ec2 -> ec2: 取得したデータをレスポンス形式に加工
            ec2 --> pwa: Response result
        deactivate

    pwa -> pwa: レスポンスデータを store に保持

    alt レスポンスデータが存在しない場合
        pwa -> user: デフォルトの負傷部位・原因登録画面を返却
    else レスポンスデータに modified の値が今月のデータが存在しない場合
        pwa -> pwa: modified の値が先月のデータの injuries.resultStatus の値が\ncontinuous のデータを抽出
        pwa --> user: 抽出したデータを負傷部位のレコードに追加した\n負傷部位・原因登録画面を返却
        pwa -> user: 抽出したデータを負傷部位のレコードに追加した\n負傷部位・原因登録画面を返却
    else 
        pwa -> pwa: modified の値が今月のデータの抽出
        pwa -> s3: 抽出したデータの　bodyImage　のリンクからデータを取得
        activate s3
            s3 --> pwa: 画像データの返却
        deactivate
        pwa --> user: 抽出したデータを負傷部位のレコードに追加、\n返却画像を追加した負傷部位・原因登録画面を返却
        pwa -> user: 抽出したデータを負傷部位のレコードに追加、\n返却画像を追加した負傷部位・原因登録画面を返却
    end

    deactivate pwa
deactivate user
@enduml
